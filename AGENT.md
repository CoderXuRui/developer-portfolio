# AGENT.md — AI 开发规范

> 本文档为 AI 助手提供项目开发指导原则。

---

## 1. 项目概述

**项目名称**：张栩睿个人技术作品集
**项目类型**：个人技术博客/作品集网站
**技术栈**：Next.js 14 + Tailwind CSS + Vercel
**部署平台**：Vercel
**主题**：浅色主题

---

## 2. 核心原则

### 2.1 开发前必读
- 优先阅读 `prd.md`（产品需求）和 `tech-design.md`（技术设计）
- 重大决策需先与用户确认，再执行
- 不确定时先问，不要猜测

### 2.2 编码规范
- **组件命名**：`PascalCase`，如 `BlogCard.tsx`
- **工具函数**：`camelCase`，如 `formatDate.ts`
- **样式**：使用 Tailwind CSS 类名，避免内联 style
- **类型**：使用 TypeScript，定义清晰接口

### 2.3 Git 规范
- 分支策略：`main`（稳定）/`dev`（开发）
- Commit 信息：简洁中文描述
- 禁止直接 push 到 main 分支

## 3. 目录结构

```
/
├── app/                    # Next.js App Router（页面）
├── components/             # React 组件
│   ├── layout/            # 布局组件
│   ├── ui/                # UI 基础组件
│   └── sections/          # 页面区块
├── content/               # Markdown 内容
│   └── blog/              # 博客文章
├── public/                # 静态资源
│   └── resume.pdf         # 简历
├── lib/                   # 工具函数
└── styles/                # 全局样式
```

---

## 4. 技术边界

### 4.1 允许做的事
- 使用 Tailwind CSS 添加/修改样式
- 创建新组件
- 优化性能
- 修复 bug
- 添加 TypeScript 类型

### 4.2 禁止做的事
- 修改 `prd.md` 和 `tech-design.md` 核心内容
- 删除用户已有内容（如简历 PDF）
- 引入未经确认的新依赖
- 破坏响应式布局

---

## 5. 工作流程

### 5.1 任务执行
1. 理解需求 → 2. 确认方案 → 3. 执行开发 → 4. 自测验证

### 5.2 修改文件前
- 先 Read 文件内容
- 确认要改的内容存在
- 使用 Edit 工具而非 Write（除非新建文件）

### 5.3 完成后
- 告知用户修改了哪些文件
- 询问是否需要测试/运行

---

## 6. 问答规范

| 场景 | 行为 |
|------|------|
| 需求不明确 | 先问用户，不要猜测 |
| 有多个方案 | 列出对比，让用户选择 |
| 发现潜在问题 | 提醒用户，共同决策 |
| 完成阶段性工作 | 汇报进展，待确认后继续 |
